# 04 — Python

This folder contains all Python work completed as part of the 
E-Commerce Sales Intelligence project. It is organised into 
two subfolders that separate the testing environment from the 
final script outputs.

---

## Files in This Folder

| File | Description |
|------|-------------|
| `README.md` | Overview of the Python folder structure and all files contained here |
| `notebooks/data_cleaning_notebook.ipynb` | Jupyter notebook used for step-by-step data cleaning, testing, and validation |
| `scripts/data_cleaning.py` | Final Python script containing the complete, tested data cleaning process |

---

## Folder Structure

### notebooks/
This subfolder contains Jupyter notebooks. A notebook allows 
code to be written and run in small individual blocks called 
cells. Each cell can be tested independently before moving on, 
which makes it the ideal environment for checking logic, 
catching errors, and documenting the reasoning behind each 
step of the analysis.

### scripts/
This subfolder contains the final Python scripts. Once the 
code inside a notebook has been tested and confirmed to work 
correctly, the clean version is transferred here as a `.py` 
script file. This represents the production-ready output of 
the Python analysis.

## Workflow

All Python work follows this two-step process throughout 
the project:

1. Code is written and tested inside the Jupyter notebook 
   in the `notebooks/` subfolder
2. Once confirmed and working, the final version is saved 
   as a clean script in the `scripts/` subfolder

This approach ensures every script in this folder has been 
fully validated before being committed to the repository.
